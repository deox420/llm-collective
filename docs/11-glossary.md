# 11 · Glosario

- **ADR** — Architecture Decision Record. Documento corto que registra una decisión de arquitectura, sus opciones y consecuencias.
- **Chairman** — modelo que produce la síntesis final.
- **Chunk** — trozo de una nota; unidad de indexado en el RAG.
- **C4** — modelo de diagramas en cuatro niveles: contexto, contenedor, componente, código.
- **Embedding** — representación vectorial de un texto, usada para búsqueda semántica.
- **MCP** — Model Context Protocol. Estándar para conectar modelos con datos y herramientas.
- **Peer review (en council)** — etapa donde cada modelo rankea las respuestas de los otros.
- **RAG** — Retrieval-Augmented Generation. El modelo recupera información de una fuente externa antes de responder.
- **Sandbox** — entorno aislado (contenedor) donde el dev-team ejecuta código sin tocar el host.
- **SSE** — Server-Sent Events. Streaming unidireccional servidor→cliente.
- **Túnel** — canal cifrado y autenticado (Tailscale/Cloudflare) para acceso remoto sin abrir puertos.
- **Vault** — carpeta de notas de Obsidian.
- **VRAM** — memoria de la GPU; determina qué modelos caben.
